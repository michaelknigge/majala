Why majala

majala is actively maintained and was initially created for an industrial strength Java application. That means that majala supports a wide range of Windows platforms, starting with Windows 2000. In fact, majala runs on every Windows system that has at least the .NET Framework 2.0 installed (of course majala works with newer versions of the .NET Framework, too). Furthermore, majala is available in 32 and 64 bit flavours.


Building

If there may be any reason to build majala by your own, the whole source code is available. The idea is that users of majala are usually Java developers that may not have much knowledge of C# or the .NET Framework. Building is quite easy because everything that is needed (i. e. third party libraries) for building is contained in the source code distribution of majala (and of course in the Subversion repositotry). Furthermore the freely availabe Visual C# Express is sufficient to build majala (the full-fledged Visual Studio is not neccessary).


Application types

majala supports various application types, i. e. console applications, graphical applications and Windows services. For every application type majala provides a separate exe-wrapper.


Bitness

For all supported application types majala provies a 32 and 64 bit exe-wrapper. Furthermore a so called AnyCPU exe-wrapper is provided that runs as a 64 bit process on a 64 bit system and as a 32 bit process on a 32 bit system.


Configuration

majala is configured with a simple line oriented configuration file (like a file containing Java properties) or with an XML file. This file can be read as a regular file from the file system or can be embedded in the majala exe-wrapper (so it may be hidden from the user).


Logging

At runtime majala uses the logging framework log4net (a port of the sophisticated log4j to the .NET Framework). In case of problems a detailed log of all majala activities, like the selection of the JVM, can be logged and analyzed. By default majala does not print any log messages, but just some simple steps are required to let majala produce a verbose log at runtime.
